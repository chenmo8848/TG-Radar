## 主要修改内容

### 1. 项目命名与入口统一

- 项目名称恢复为 **TG-Radar**
- 默认部署目录改为 **`/root/TG-Radar`**
- 全局控制命令改为 **`TR`**
- 终端、安装向导、README、Telegram 面板口径全部统一

### 2. Linux 安装与管理界面重写

- `install.sh`
  - 改成统一的一键部署入口
  - 本地执行与 `bash <(curl ...)` 远程执行共用同一套逻辑
  - 配置向导文案重写
  - 输出更现代、更完整、更像正式项目安装器

- `deploy.sh`
  - 终端控制中心围绕 `TR` 重写
  - 状态、自检、日志、更新、卸载的说明更明确
  - 旧版命令与旧版服务残留清理更彻底

### 3. Telegram ChatOps 交互全面升级

- `help` 面板重写
- `status` 监控大屏重写
- `config` 配置面板重写
- `sync` 同步结果面板重写
- `update` / `restart` 提示文案重写
- `folders` / `rules` / `routes` 输出更详细

### 4. 编辑现有消息与垃圾消息自动回收

这一版专门补强了这条体验链：

- 优先尝试 **编辑你刚刚在收藏夹发出的命令消息**
- 如果因为 Telegram 条件限制无法编辑，就 fallback 为回复
- fallback 回复会自动回收
- 原始命令消息在 fallback 场景下也会自动清理
- 帮助面板、状态面板、同步面板会按设定时间自动删除

### 5. 告警与启动提示优化

- 告警消息内容更完整：
  - 命中词
  - 命中规则
  - 所属分组
  - 捕获时间
  - 来源聊天
  - 发送者
  - Chat ID / Message ID
  - 原始消息快照
  - 直达链接（如果能生成）

- 启动通知更完整：
  - 架构
  - 启动时间
  - 自动同步频率
  - 热更新频率
  - 当前监听规模
  - 自动收纳规则
  - 版本号

### 6. 运行目录与仓库结构收尾

- 运行期文件全部收口到 `runtime/`
- 增加 `runtime/README.md`
- `.gitignore` 调整为忽略 runtime 真数据，但保留目录说明文件
- 清掉了不该继续混在仓库里的隐藏占位文件和缓存文件
